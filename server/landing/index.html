LPR Server

AI Camera – ศูนย์รวมข้อมูลและสถานะระบบ

สถาปัตยกรรมและวิธีการทำงาน

ระบบแบ่งเป็น เครื่องลูกข่าย (Edge) และ เครื่อง Server นี้. ลูกข่ายส่งข้อมูล (ข้อความ, ภาพ) ผ่าน WebSocket หรือ MQTT เข้ามาที่ Microservices บนเครื่องนี้. ข้อมูลถูกจัดเก็บหรือประมวลผลแล้วนำไปแสดงในส่วนแสดงผลจากฐานข้อมูล.

# Client (Browser / Edge device)
/ → Landing (หน้านี้)
/server/ → Detection Dashboard (ข้อมูลจาก DB + สถานะ Services)
/server/network → Communication Network (สถานะ API, WebSocket, MQTT)
/server/edge_control → Edge AI Dashboard (สถานะอุปกรณ์ Edge)
/ws/ → WebSocket (Socket.IO) สำหรับลูกข่าย (ผ่าน Nginx port 80)
# Microservices (บนเครื่องนี้)
backend-api (port 3000) – REST API, อ่าน/เขียน DB
ws-service (port 3001 ภายใน) – รับข้อความ/ภาพผ่าน Socket.IO
mqtt-service – รับข้อมูลผ่าน MQTT broker
frontend – แสดงผลที่ /server/

เมนูเข้าถึงระบบ

  • Server (Detection Dashboard) – แสดงผลการตรวจจับยานพาหนะจาก Edge AI Client และสถานะ Services
  • Network (Communication Network) – แสดงสถานะ Microservice (REST API, WebSocket, MQTT) และข้อมูลสำหรับ Config Client
  • Edge Control (Edge AI Dashboard) – แสดงสถานะ Edge Client Device (แบตเตอรี่, ดิสก์, CPU, RAM, อุณหภูมิ, uptime, พิกัด, เครือข่าย) จาก MQTT

การเชื่อมต่อสำหรับลูกข่าย

ลูกข่าย (Edge) สามารถส่งข้อมูลเข้ามาได้ผ่าน:

  • WebSocket (Socket.IO): ใช้ URL http://lprserver.tail605477.ts.net/ws/ (path: /ws/)
  • MQTT: ตามการตั้งค่า MQTT broker และ topic ที่ Microservice ใช้

รายละเอียดการส่งข้อความและภาพผ่าน WebSocket อยู่ใน Server และเอกสาร WebSocket Client Guide ในโปรเจกต์
