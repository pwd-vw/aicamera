# Manual Capture System

ระบบการบันทึกภาพด้วยตนเองสำหรับ AI Camera Dashboard

## 📁 โครงสร้างโฟลเดอร์

```
/edge/manual_capture/
├── README.md                    # ไฟล์นี้
├── manual_capture_YYYYMMDD_HHMMSS_mmm.jpg  # ภาพที่บันทึก
└── ...
```

## 🎯 การใช้งาน

### 1. การบันทึกภาพผ่าน Dashboard
- เปิดหน้า Camera Dashboard
- คลิกปุ่ม **"Capture"** (📷) ที่อยู่ใต้ video feed
- ระบบจะบันทึกเฟรมปัจจุบันและแสดงข้อความยืนยัน
- ภาพจะถูกบันทึกในโฟลเดอร์นี้โดยอัตโนมัติ

### 2. รูปแบบชื่อไฟล์
```
manual_capture_YYYYMMDD_HHMMSS_mmm.jpg
```
- `YYYYMMDD`: วันที่ (ปี-เดือน-วัน)
- `HHMMSS`: เวลา (ชั่วโมง-นาที-วินาที)
- `mmm`: มิลลิวินาที (3 หลัก)

### 3. คุณภาพภาพ
- ความละเอียด: ตามการตั้งค่ากล้องปัจจุบัน
- รูปแบบ: JPEG (.jpg)
- คุณภาพ: 95% (สูงสุด)
- การบีบอัด: น้อยที่สุดเพื่อรักษาคุณภาพ

## 🔧 การตั้งค่า

### โฟลเดอร์การบันทึก
- ตำแหน่ง: `/edge/manual_capture/`
- สิทธิ์: อ่าน/เขียนสำหรับผู้ใช้ระบบ
- การสร้าง: สร้างอัตโนมัติเมื่อเริ่มต้นระบบ

### การตั้งค่ากล้อง
- ระบบจะใช้การตั้งค่ากล้องปัจจุบัน
- รองรับการปรับแต่ง brightness, contrast, saturation
- รองรับการตั้งค่า white balance (AWB)

## 📊 ข้อมูลภาพ

เมื่อบันทึกภาพสำเร็จ ระบบจะแสดงข้อมูล:
- ชื่อไฟล์
- ขนาดไฟล์ (bytes)
- ความละเอียด (width x height)
- เวลาที่บันทึก
- สถานะการบันทึก

## 🚨 การแก้ไขปัญหา

### ปัญหาที่พบบ่อย

1. **ไม่สามารถบันทึกภาพได้**
   - ตรวจสอบว่ากล้องทำงานอยู่
   - ตรวจสอบสิทธิ์การเขียนในโฟลเดอร์
   - ตรวจสอบพื้นที่ว่างในดิสก์

2. **ภาพไม่ชัดเจน**
   - ปรับการตั้งค่ากล้อง (brightness, contrast, saturation)
   - เปลี่ยน white balance mode
   - ตรวจสอบการโฟกัสของกล้อง

3. **ไฟล์เสียหาย**
   - ตรวจสอบการเชื่อมต่อกล้อง
   - รีสตาร์ทระบบ
   - ตรวจสอบ log files

### การตรวจสอบ Log
```bash
# ตรวจสอบ log ของระบบ
tail -f /home/camuser/aicamera/edge/logs/camera.log

# ตรวจสอบ log ของ web service
tail -f /home/camuser/aicamera/edge/logs/web.log
```

## 📝 หมายเหตุ

- ระบบจะบันทึกภาพจากเฟรมปัจจุบันของ video stream
- ภาพจะถูกบันทึกทันทีเมื่อคลิกปุ่ม Capture
- ไม่มีการบีบอัดเพิ่มเติมเพื่อรักษาคุณภาพ
- ระบบรองรับการบันทึกต่อเนื่อง (ไม่มีการจำกัดจำนวน)
